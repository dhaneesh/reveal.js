Pre Drupal 8

  • PHPTemplate theme engine
  • PHP in template (.tpl.php) files
  • Not secure but easy to debug

Debugging PHPEngine Template

<?php print_r($my_variable); ?>
<?php var_dump($my_variable); ?>
<?php dpm($my_variable); ?>

Problem!

Have to add additional code for debugging.

Developers are forgetful.

Solution: Xdebug

  • Debugger & Profiler tool for PHP
  • Plugin for many IDEs (PHPStorm, Sublime Text, eclipse, etc...)
  • Can use breakpoints instead of code to debug

Xdebug Breakpoint in PHPStorm

Drupal 8 and Beyond

  • Twig Template theme engine
  • No PHP allowed in template (.html.twig) files.
  • More secure, easy to debug

Debugging Twig Template

{{ dump(my_variable) }}
{{ kint(my_variable) }}
???

Same Problem!

Have to add additional code for debugging.

Developers are forgetful.

Breakpoints?

  • No support for breakpoints in twig
  • Can create a Twig Extension
  • Still requires additional code

What I have learned and hate about Drupal 8...

No way to debug Twig templates without adding additional code.

Totally Worth It

Security & Performance > Lazyness
